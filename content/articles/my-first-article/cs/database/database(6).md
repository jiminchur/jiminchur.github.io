### 1. 관계 대수(Relational Algebra) 연산

관계 대수는 관계형 데이터베이스의 질의를 수학적으로 표현하는 도구로, 여러 기본 연산자들이 있습니다.

- **단항 연산**
    - **SELECT (σ):**
        - 조건을 만족하는 투플(행)을 선택합니다.
        - 예: σ<sub>조건</sub>(R)
    - **PROJECT (π):**
        - 특정 속성(열)만 추출하여 새로운 릴레이션을 생성합니다.
        - 예: π<sub>속성리스트</sub>(R)
    - **RENAME (ρ):**
        - 릴레이션 혹은 속성의 이름을 재지정합니다.
- **집합 연산**
    - **합집합 (∪), 교집합 (∩), 차집합 (-):**
        - 두 릴레이션이 ‘합집합 호환(Union Compatible)’일 때 적용됩니다.
    - **카티션 곱 (×):**
        - 두 릴레이션의 모든 투플 조합을 생성합니다.
- **조인 연산**
    - **세타 조인 (θ-join):**
        - 일반 조건(예: <, >, = 등)을 사용하여 두 릴레이션을 결합합니다.
    - **동등 조인 (Equi-join):**
        - "=" 조건만 사용한 조인으로, 조건에 사용된 속성의 중복이 나타날 수 있습니다.
    - **자연 조인 (Natural Join):**
        - 공통 속성을 기준으로 결합하며 중복된 속성을 하나로 제거합니다.
    - **자체 조인 (Self Join):**
        - 동일한 릴레이션의 서로 다른 사본을 이용해 자기 자신과 조인합니다.
- **디비젼 연산 (Division):**
    - “모든 y에 대해 조건을 만족하는 x”를 찾는 연산으로, 특정 속성 집합의 모든 값을 갖는 x값들을 구할 때 사용합니다.
- **추가 연산**
    - **집계 함수 및 그룹화:**
        - SUM, COUNT, AVG, MIN, MAX 등의 함수를 사용해 그룹별 집계 처리가 가능합니다.
    - **외부 조인 및 외부 합집합:**
        - 조인 조건을 만족하지 않는 투플도 포함하는 연산으로, LEFT, RIGHT, FULL OUTER JOIN 등이 있습니다.

### 2. SQL 기본 질의와 데이터 조작

SQL은 관계형 데이터베이스에서 데이터를 정의, 조작, 제어하기 위한 표준 언어입니다. 6주차에서는 SQL의 기본 질의 작성과 DML(Data Manipulation Language) 명령어들을 다룹니다.

- **SQL 질의문의 기본 구조**
    - **SELECT:** 출력할 속성(열)을 지정
    - **FROM:** 데이터를 추출할 테이블(들)을 명시
    - **WHERE:** 조건에 따라 투플(행)을 선택
    - **그룹화 및 집계:**
        - **GROUP BY:** 동일한 값을 가지는 투플들을 그룹으로 묶음
        - **HAVING:** 그룹 단위 조건을 지정
    - **ORDER BY:** 결과 정렬(ASC/DESC)
- **집합 연산**
    - SQL은 **UNION**, **INTERSECT**, **EXCEPT** 명령어를 통해 집합 연산을 지원합니다.
    - 기본적으로 결과는 다중집합(Multiset 또는 Bag)으로 반환되며, **DISTINCT** 키워드로 중복을 제거할 수 있습니다.
- **중첩 질의(Nested Query)**
    - 서브쿼리(중첩된 SELECT문)를 WHERE 절이나 FROM 절에 포함시켜 복잡한 조건을 처리할 수 있습니다.
- **데이터 조작 명령(DML)**
    - **INSERT:** 테이블에 새로운 투플을 추가
    - **DELETE:** 특정 조건에 맞는 투플들을 삭제
    - **UPDATE:** 특정 조건에 맞는 투플들의 값을 수정
    - 이들 명령어 실행 시, 테이블에 정의된 무결성 제약조건(예: 키, 참조 무결성 등)이 반드시 유지되어야 합니다.
- **뷰(View)의 활용**
    - 뷰는 여러 테이블의 조인 결과나 복잡한 질의를 하나의 가상 테이블로 정의하여 자주 사용하는 데이터를 편리하게 조회할 수 있도록 도와줍니다.
    - **CREATE VIEW** 구문으로 정의하며, 기본 테이블의 최신 데이터를 동적으로 반영합니다.

### 3. 6주차 학습 포인트

- **관계 대수 연산:**
    - 각 연산자의 기능과 연산자들을 조합하여 복잡한 질의를 구성하는 방법을 연습합니다.
- **SQL 기본 질의 작성:**
    - SELECT 문의 기본 구조와 집합 연산, 중첩 질의, 그룹화 및 정렬 기능 등을 숙지합니다.
- **데이터 조작과 무결성:**
    - INSERT, DELETE, UPDATE 문을 통한 데이터 조작 시, 무결성 제약조건이 어떻게 적용되는지 이해합니다.
- **뷰의 개념 및 활용:**
    - 뷰를 사용해 복잡한 조인을 단순화하고, 재사용 가능한 질의 템플릿을 구성하는 방법을 학습합니다.
