### 데이터베이스의 정의와 개념

- **데이터베이스(Database)**
    - 한 조직의 업무에 필요한 상호 관련된 데이터를 중복 없이 저장·관리하는 집합체
    - “작은 세계”나 “논의 세계”라 불리는 실세계의 일부분을 표현하는 데이터 모임
- **데이터의 종류**
    - **통합 데이터:** 여러 응용 프로그램이 공동으로 사용하는 데이터
    - **저장 데이터:** 컴퓨터의 기억장치에 저장된 자료
    - **운영 데이터:** 조직의 핵심 업무 수행에 필수적인 데이터
    - **공용 데이터:** 여러 사용자 및 응용 프로그램이 공유하는 데이터

---

### 2. 데이터베이스 시스템(DBMS)의 개요

- **DBMS의 역할**
    - 데이터베이스 생성, 관리, 조작, 보호 및 유지보수를 담당하는 소프트웨어
    - 사용자에게 데이터 정의, 질의 처리, 갱신 및 백업/회복 기능 제공
- **DBMS의 주요 기능**
    - **데이터 정의:** 데이터 타입, 구조, 제약조건 등을 명세
    - **데이터 조작:** 질의, 갱신, 보고서 생성을 위한 기능
    - **동시성 제어 및 보안:** 다수 사용자가 동시에 접근할 때 데이터의 일관성 유지 및 무단 접근 방지
    - **데이터 독립성:** 응용 프로그램과 데이터의 저장 방식이 분리되어 있어 구조 변경 시 영향 최소화

---

### 3. 데이터베이스 시스템의 특징 및 장점

- **주요 특징**
    - **실시간 접근성:** 비정형 질의에 대해 빠른 응답 제공
    - **계속적인 변화:** 새로운 데이터의 삽입, 삭제, 갱신을 통해 항상 최신 데이터 유지
    - **동시공용:** 여러 사용자가 동시에 데이터에 접근 가능
    - **내용에 의한 참조:** 데이터의 위치가 아닌 의미를 바탕으로 데이터 검색
- **DBMS 사용의 장점**
    - **데이터 중복 최소화:** 불필요한 데이터 중복으로 인한 저장 공간 낭비와 갱신 문제 해소
    - **데이터 무결성 유지:** 변경 시 모든 관련 데이터를 자동으로 동기화
    - **보안 및 접근 제어:** 사용자별 권한 부여를 통한 데이터 보호
    - **표준화 및 통합 관리:** 여러 응용 프로그램 간 데이터 일관성을 유지하며 효율적인 관리 가능
- **파일 시스템 대비 DBMS의 단점**
    - 초기 전산화 비용 증가 및 DB 전문가의 필요성
    - 복잡한 시스템으로 인한 백업, 회복 절차의 어려움

---

### 4. 데이터베이스 사용자 분류

- **데이터베이스 관리자(DBA):** 시스템 운영, 유지보수, 성능 튜닝 등을 담당
- **데이터베이스 설계자:** 데이터베이스의 구조와 스키마 설계
- **최종 사용자(End Users):** 질의, 데이터 갱신, 보고서 작성 등을 수행
    - **캐주얼 사용자:** 비정기적 데이터 접근
    - **초보 사용자:** 미리 작성된 프로그램 사용
    - **전문 사용자:** 복잡한 응용 프로그램 개발 및 DBMS 기능 활용
- **시스템 분석가/응용 프로그래머:** 사용자 요구에 맞는 응용 프로그램 설계 및 구현
