### 1. 관계형 모델(Relational Model)의 기본 개념

- **릴레이션(테이블)과 투플(행), 애트리뷰트(열)**
    - 릴레이션은 데이터베이스의 기본 단위로, 여러 투플(행)의 집합입니다.
    - 각 투플은 여러 애트리뷰트(열)로 구성되며, 각 애트리뷰트는 미리 정의된 도메인(값의 집합)에 속합니다.
- **키(Key)와 도메인**
    - **도메인 제약:** 각 애트리뷰트에 입력되는 값은 해당 도메인에 속해야 합니다.
    - **키 제약:** 각 테이블에는 투플들을 유일하게 식별할 수 있는 키(예: 후보키, 주키, 대체키, 슈퍼키)가 있어야 하며, 키 값은 중복되거나 NULL이 될 수 없습니다.
    - **외래키(Foreign Key):** 한 테이블의 외래키는 다른 테이블의 기본키를 참조하여 두 테이블 간의 관계를 표현합니다.

---

### 2. 관계 모델의 무결성 제약조건

- **도메인 무결성:**
    - 각 속성의 값이 미리 정의된 도메인의 원소여야 합니다.
- **키 무결성:**
    - 기본키(주키)는 유일해야 하며, NULL 값을 가질 수 없습니다.
- **참조 무결성:**
    - 외래키는 참조하는 테이블의 기본키에 존재하는 값이어야 하며, 그렇지 않을 경우 제약 조건 위반으로 처리됩니다.
- **비즈니스 룰/시멘틱 제약조건:**
    - 조직 특유의 규칙(예: 사원의 봉급이 특정 범위를 넘어서는 안 된다 등)을 추가적으로 설정할 수 있습니다.

---

### 3. 무결성 제약조건 위반 처리 및 갱신 연산

- **삽입(Insert) 연산 시**
    - 입력된 데이터가 도메인, 키, 참조 제약조건을 만족하지 않으면 삽입이 거부됩니다.
    - 예를 들어, 이미 존재하는 키 값을 삽입하거나, 외래키가 참조하는 값이 없으면 오류가 발생합니다.
- **삭제(Delete) 연산 시**
    - 삭제할 투플이 다른 테이블의 외래키로 참조되고 있다면, 참조 무결성 제약조건 위반이 발생합니다.
    - 이를 해결하기 위해 삭제를 거부하거나, CASCADE, SET NULL, SET DEFAULT 옵션 등을 사용해 참조 관계를 처리할 수 있습니다.
- **갱신(Update) 연산 시**
    - 기본키나 외래키 변경 시에도 무결성 제약조건을 준수해야 하며, 갱신 연산은 삭제 후 삽입과 유사하게 처리될 수 있습니다.
- **실제 사례 예제**
    - 예를 들어, 부서 테이블에서 부서번호가 기본키인데 NULL로 삽입되는 경우 엔티티 무결성 제약 조건을 위반하게 됩니다.
    - 또는, EMP 테이블에서 존재하는 사원번호와 중복되는 사원번호로 삽입하려 하면 키 제약 조건 위반이 발생합니다.

### 4. 학습 포인트

- **핵심 용어 복습:**
    - 릴레이션, 투플, 애트리뷰트, 도메인, 키, 외래키 등 기본 개념을 명확히 이해합니다.
- **무결성 제약조건의 역할:**
    - 각 제약조건(도메인, 키, 참조, 비즈니스 룰)이 데이터의 정확성과 일관성을 어떻게 보장하는지 이해합니다.
- **연산과 무결성의 상관관계:**
    - 삽입, 삭제, 갱신 시 발생할 수 있는 무결성 위반 사례와 그 처리 방법(예: CASCADE, SET NULL 등)을 예제를 통해 점검합니다.
