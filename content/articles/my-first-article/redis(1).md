## 인메모리 저장소와 Redis 소개
Redis에 대해서 자세하게 알아보기전 인메모리 저장소의 필요성에 대해서 한번 알아보겠다.

### 인메모리 저장소의 필요성
우리는 Spring Boot를 사용하면서 RDB - 관계형 데이터베이스를 사용하였다. 관계형 데이터베이스의 주된 목적은 영속성과 일관성이라고 얘기할 수 있으며 즉 파일시스템에 데이터를 저장함으로써 서비스가 종료가 되어도 유지되는 데이터를 위해서 이다.

하지만 서비스를 만들다보면 일시적인 데이터를 저장해야 하는 상황이 빈번히 발생하게 되다. 예를들면 로그인 정보, 쿠키나 세션, 장바구니와 같은 기능은 사용자의 행동에 따라 빈번하게 데이터의 수정이 발생하게 된다.

이러한 상황에서 파일시스템에 데이터를 저장하는 것은 그 특성상 속도가 느릴 수 밖에 없다. 그래서 특정 기능의 구현을 위해서 파일 시스템을 사용하는 관계형 데이터베이스 대신 메모리를 사용하여 데이터를 일시적으로 보관하는 데이터베이스를 찾아볼 수 있다.

## Redis란 무엇일까??
> 레디스는 Remote Dictionary Server의 약자로서, "키-값" 구조의 비정형 데이터를 저장하고 관리하기 위한 오픈 소스 기반의 비관계형 데이터베이스 관리 시스템이다. 2009년 살바토르 산필리포가 처음 개발했다. 2015년부터 Redis Labs가 지원하고 있다
구글에 "Redis란?"이라고 검색을 하게되면 위와 같이 나온다. Java의 Map과 같은 방식으로 데이터를 저장하는 데이터베이스이다. 

### Redis의 특징
일반적인 관계형 데이터베이스와 다른 가장 큰 특징은 In-Memory 데이터베이스라는 특징이다. 파일 시스템에 데이터를 변경하는건 느리지만 메모리의 데이터를 변경하는건 상대적으로 빠르다. 즉, RAM에 데이터를 저장하기 때문에 복잡한 입출력 과정이 필요하지 않는다. 한문장으로 얘기를 해보면 관계형 데이터베이스에 비해 더 빠르게 동작하는 대신, 언제든 사라질 수 있는 데이터를 다룬다라고 말을 할 수가 있다. 
### 주로 어디서 사용되는가??? 
Redis는 특정 게시글의 조회수와 같이 빠르게 업데이트되ㅣ는 데이터 또는 사용자 세션 장바구니 와 같은 시간이 지나면 삭제되는 데이터등을 저장하기 위해서 가장 많이 사용되어온 데이터베이스이다.

## NoSQL이란??
> 개요[편집] 단어 뜻 그 자체를 따지자면 "Not only SQL"로, SQL만을 사용하지 않는 데이터베이스 관리 시스템(DBMS)을 지칭하는 단어이다. 관계형 데이터베이스를 사용하지 않는다는 의미가 아닌, 여러 유형의 데이터베이스를 사용하는 것이다.
구글에 검색을 해보면 위와 같이 나오는 걸 확인 할 수가 있다. 즉 SQL을 사용하지 않는다라고 알아두면 좋을 거 같다. 이들은 일반적인 관계형 데이터베이스가 약점을 가진 확장성,유연성,성능에 대한 문제를 해결하기 위해 사용되는 경우가 많으며, 컴퓨터 기술의 발전과 웹의 활성화로 인해 비정형 데이터를 더 높은 성능으로 사용하는데 초점이 맞춰진 경우가 많다.

### NoSQL 사용법
* Key-Value
  * 가장 단순한 형태의 데이터베이스로, Key에 Value를 저장하는 형태입니다. JSON, Python의 Dictionary, Java의 Map의 형태로 데이터를 관리한다고 생각할 수 있습니다.
* Document
  * 객체를 표현하는 Document라는 단위로 데이터를 저장하는 형태입니다. Key - Value에서 발전했다고 볼 수 있으며, JSON, XML 등 복잡한 데이터를 저장하고 관리합니다.
* Column-Family
  * 각 Row의 Column이 고정되어있지 않고, 필요한 데이터 Column을 이름, 데이터, Timestamp와 함께 저장하는 형태의 데이터베이스 입니다.

Key-Value는 Redis, Document는 MongoDB, Column-Family는 Cassandra로 대표적인 예시를 얘기할 수 있다.

## Redis의 활용 사례
Redis는 Key-Value Store로 작동하는 인메모리 데이터베이스이다. 지연이 적은 읽기/쓰기 성능을 가졌으며 그렇게 때문에 일시적인 데이터,변경이 잦은 데이터를 다뤄야 되는 상황에서 많이 활용된다.

* Session Clustering
  * 여러 애플리케이션 인스턴스에서 같은 세션 정보를 사용할 수 있도록 도와준다.
* Caching
  * 자주 사용되는 데이터를 저장해두어, 데이터베이스 조회를 줄이고 전반적인 응답속도를 개선한다.
* 지원하는 다양한 자료구조를 바탕으로 리더보드, 방문수 트래킹, 좌표 기반 검색 등의 기능을 쉽게 구현할 수 있게 해준다.
