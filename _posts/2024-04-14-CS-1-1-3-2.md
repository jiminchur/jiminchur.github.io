### 파이썬 코드의 결과는 무엇일까??
```
a = 0.1
b = 0.2
c = 0.3

if a + b == c:
    print("같다")
else :
    print("같지않다.")
```
정답은 "같지않다" 이다.

### 왜 이런 결과가 나온 걸까??
* 컴퓨터 내의 소수점 표현 방식의 한계 때문
* 부동 소수점(floating point)의 한계 때문

### e.g) 123.123 심진수를 m X 10^n 꼴로 나타낸다고 가정
* 1.23123 X 10^2
* 1231.23 X 10^-1

여기서
* 빨간 글씨 = 지수 (exponent)
* 파란 글씨 = 가수 (significand)


### 컴퓨터 내부의 소수 표현 방식(부동 소수점 표현 방식)
* 이진수를 m X 2^n 꼴로 나타내고, 가수 부분은 1.XXX 꼴로 띄고 있음
    * XXX 부분(소수 부분)만 저장하면 됨
    * 바이어스(bias) 값이 더해져 저장됨. 바이어스 값은 2^(k-1)-1(k는 지수의 비트 수)

### 이와 같은 표현의 문제
* 십진수 소수를 이진수로 표현할 때 십진수 소수와 이진수 소수 표현이 딱 맞아 떨어지지 않을 수 있다.
* e.g 1/3이라는 분수 m X 3^n 꼴로 표현하면 딱 떨어진다.
* e.g 1/3이라는 분수 m X 10^n 꼴로 표현하면 무한히 많은 가수가 필요하다.
