[Algorithm] 순차 탐색과 이진 탐색 트리

## **1️⃣ 트리 탐색 (Tree Traversal)** ### **트리 순회 (Tree Traversal)** 트리 탐색은 트리 내의 모든 노드를 정확히 한 번만 방문하는 과정입니다. 트리 구조에서 각 노드를 어떻게 방문할지에 따라 다양한 순회 방법이 있습니다. ### **트리 순회의 종류** 1️⃣ **전위 순회 (Preorder Traversal)** - 순서: **루트 → 왼쪽 서브트리 → 오른쪽 서브트리**- **예제**: 루트 노드를 먼저 방문하고, 왼쪽과 오른쪽 서브트리를 순회합니다. 📌 **전위 순회 알고리즘** ```pythondef preorder_traversal(node): if node: print(node.value, end=" ") preorder_traversal(node.left) preorder_traversal(node.right)``` 2️⃣ **중위 순회 (Inorder Traversal)** - 순서: **왼쪽 서브트리 → 루트 → 오른쪽 서브트리**- **특징**: 이진 탐색 트리에서 중위 순회를 수행하면 결과가 오름차순으로 나옵니다. 📌 **중위 순회 알고리즘** ```pythonpython복사편집def inorder_traversal(node): if node: inorder_traversal(node.left) print(node.value, end=" ") inorder_traversal(node.right) ``` 3️⃣ **후위 순회 (Postorder Traversal)** - 순서: **왼쪽 서브트리 → 오른쪽 서브트리 → 루트**- **예제**: 두 자식 노드를 모두 방문하고, 마지막으로 루트 노드를 방문합니다. 📌 **후위 순회 알고리즘** ```pythondef postorder_traversal(node): if node: postorder_traversal(node.left) postorder_traversal(node.right) print(node.value, end=" ")``` ### ✅ **중요 포인트** - **전위, 중위, 후위 순회**는 트리에서 데이터를 방문하는 순서가 다릅니다.- *이진 탐색 트리(BST)**에서 중위 순회는 오름차순 정렬을 보장합니다. --- ## **2️⃣ 이진 탐색 트리 (Binary Search Tree, BST)** ### **이진 탐색 트리란?** - **이진 탐색 트리**는 각 노드가 최대 두 개의 자식을 가지며, 왼쪽 서브트리의 값은 현재 노드의 값보다 작고, 오른쪽 서브트리의 값은 더 큰 트리 구조입니다.- 삽입, 삭제, 탐색을 효율적으로 수행할 수 있는 자료구조입니다. 📌 **이진 탐색 트리 예시** ``` 23 / \ 11 39 / \ / \ 1 13 31 45 ``` ### ✅ **이진 탐색 트리 특징** - **검색**: O(log n) (균형 잡힌 경우)- **삽입/삭제**: O(log n) (균형 잡힌 경우)- **효율적인 탐색**을 위해 트리의 균형을 유지하는 것이 중요합니다. --- ## **3️⃣ 이진 탐색 트리의 삽입 (Insertion)** ### **삽입 과정** 1️⃣ **루트 노드**부터 시작하여 값이 작은 경우 왼쪽 서브트리로, 큰 경우 오른쪽 서브트리로 삽입합니다.2️⃣ 값을 찾은 위치에 **새로운 노드**를 삽입합니다. 📌 **이진 탐색 트리 삽입 예시** - **입력 값**: 15, 8, 25, 18 트리 구조: ``` 15 / \ 8 25 / 18 ``` 📌 **이진 탐색 트리 삽입 코드** ```pythondef insert_bst(root, key): if root is None: return TreeNode(key) if key < root.value: root.left = insert_bst(root.left, key) else: root.right = insert_bst(root.right, key) return root ``` --- ## **4️⃣ 트리 탐색과 삽입** ### **트리 탐색과 삽입을 동시에 다룰 때 유의할 점** - **탐색**: 트리 구조에서 효율적으로 원하는 노드를 찾을 수 있습니다.- **삽입**: 새로운 데이터를 추가할 때마다 트리의 균형을 고려해야 합니다. ✅ **이진 탐색 트리의 장점**: - 데이터가 많아도 **탐색 효율성**이 좋음- 삽입 및 삭제도 효율적으로 처리 가능 --- ## **5️⃣ 위상 정렬 (Topological Sort)** ### **위상 정렬이란?** - **위상 정렬**은 **비순환 방향 그래프(DAG)**에서 각 작업의 선후 관계를 고려하여 작업을 나열하는 정렬 방법입니다.- **선행 작업을 먼저** 처리하고, 그 다음에 후속 작업을 처리하는 방식으로 여러 작업을 순차적으로 처리할 수 있게 합니다. 📌 **위상 정렬 예시** - 여러 **작업**이 있고, 각 작업 사이에 **선후 관계**가 있을 때, 각 작업의 순서를 나열하는 문제입니다. 📌 **위상 정렬의 시간 복잡도**: O(V + E) (V = 정점의 수, E = 간선의 수)
