알고리즘이란?

  • 알고리즘 (Algorithm):

    • 문제를 해결하는 절차 또는 과정

    • 입력을 받아 원하는 출력을 만들어내는 규칙의 집합

    • 순서도, 의사 코드, 프로그래밍 언어로 표현 가능

✅ 알고리즘의 예시

  • 요리 🍳 → 레시피 = 요리 알고리즘

  • 게임 🎮 → 미로를 빠르게 탈출하는 방법

  • 네비게이션 🛣 → 최단 경로 찾기

  • ATM 🏦 → 스케줄링 알고리즘 사용

  • 인터넷 검색 🔎 → 검색 최적화 알고리즘

알고리즘의 작성 과정

💡 문제 해결을 위한 7단계

  1. 문제 분석 🧐 → 핵심 사항을 파악

  2. 데이터 수집 & 표현 📊 → 필요한 정보 정리

  3. 문제 분해 🧩 → 복잡한 문제를 작은 단위로 나눔

  4. 패턴 인식 🔄 → 문제 속 공통된 규칙 찾기

  5. 추상화 🎭 → 불필요한 요소 제거

  6. 알고리즘 설계 🏗 → 해결책을 단계적으로 정리

  7. 평가 ✅ → 정확성과 효율성 점검 후 코딩 시작!

✨ 예제: 최고점 찾기

  • 📌 학생 10명의 점수 중 최고점을 찾는 알고리즘

    1. 첫 번째 점수를 저장한다.

    2. 다음 점수를 읽어 현재 저장된 점수와 비교한다.

    3. 더 큰 점수를 저장한다.

    4. 모든 점수를 비교한 후 최고 점수를 출력한다.

  • 📌 의사 코드 (Pseudocode)

max = A[0]

for i in range(1, 10):

if A[i] > max:

max = A[i]

return max

알고리즘의 특성

  • 📌 알고리즘은 다음 조건을 만족해야 한다!

    • ✅ 입력(Input) → 입력 값이 주어져야 한다.

    • ✅ 출력(Output) → 반드시 결과(출력)가 있어야 한다.

    • ✅ 유한성(Finiteness) → 반드시 끝나야 한다.

    • ✅ 정확성(Correctness) → 올바른 답을 도출해야 한다.

    • ✅ 일반성(Generality) → 다양한 문제에도 적용 가능해야 한다.

4️⃣ 알고리즘의 표현 방법

✍ 알고리즘을 표현하는 3가지 방법 1️⃣ 순서도 (Flow Chart) - 도형과 기호를 이용한 다이어그램 📊2️⃣ 의사 코드 (Pseudocode)- 사람이 이해하기 쉬운 코드 형태 💻3️⃣ 일반 언어 표현- 자연어로 서술 ✍ --- 5️⃣ 알고리즘의 수행 시간 분석 🔍 효율적인 알고리즘 선택이 중요한 이유! - 같은 문제라도 알고리즘에 따라 수행 시간이 천차만별 ⚡- 실행 속도를 비교할 때 Big-O 표기법 사용 📌 알고리즘의 시간 복잡도 ```O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) ``` 📌 예제 - `O(1)` → 배열의 특정 인덱스 값 가져오기- `O(n)` → 모든 배열 요소의 합 구하기- `O(n²)` → 이중 루프 사용 (예: 선택 정렬) --- 6️⃣ 정렬 알고리즘 📌 정렬이란? → 데이터를 오름차순 또는 내림차순으로 배열하는 것 🔹 기본적인 정렬 알고리즘 1. 선택 정렬 (Selection Sort) - O(n²) - 가장 작은 값을 찾아 차례대로 정렬 - 예제: 키 작은 사람부터 줄 세우기2. 버블 정렬 (Bubble Sort) - O(n²) - 이웃한 두 숫자를 비교하며 정렬 - 예제: 가벼운 공이 위로 뜨는 원리3. 삽입 정렬 (Insertion Sort) - O(n²) → 최선 O(n) - 새로운 데이터를 적절한 위치에 삽입 - 예제: 카드를 정리하는 방식4. 병합 정렬 (Merge Sort) - O(n log n) - 데이터를 반으로 나누고 정렬 후 합치는 방식 - 예제: 그룹별 조를 나눈 뒤 합치기5. 퀵 정렬 (Quick Sort) - O(n log n) - *기준(pivot)을 정하고, 작은 값은 왼쪽, 큰 값은 오른쪽으로 정렬 --- 7️⃣ 탐색 알고리즘 📌 탐색이란? → 데이터 속에서 원하는 값을 찾는 것 🔹 대표적인 탐색 알고리즘1️⃣ 순차 탐색 (Sequential Search) - O(n) - 처음부터 하나씩 비교하며 찾기- 예제: 사전에서 단어 찾기 (A부터 하나씩 살펴보기) 2️⃣ 이진 탐색 (Binary Search) - O(log n) - 정렬된 데이터에서만 가능!- 중간값과 비교하며 범위를 반씩 줄여가는 방식- 예제: 전화번호부에서 특정 이름 찾기 📖 3️⃣ 이진 탐색 트리 (BST) - O(log n) - 탐색, 삽입, 삭제가 효율적인 트리 자료구조 4️⃣ 레드-블랙 트리 (Red-Black Tree) - O(log n) - 자동으로 균형을 유지하는 이진 탐색 트리- 검색 엔진, 데이터베이스에 활용 | 정렬 알고리즘 | 시간 복잡도 | 특징 || --- | --- | --- || 선택 정렬 | O(n²) | 가장 작은 값부터 선택 || 버블 정렬 | O(n²) | 인접한 값끼리 비교 || 삽입 정렬 | O(n²), 최선 O(n) | 데이터를 삽입하며 정렬 || 병합 정렬 | O(n log n) | 안정적인 정렬 (많이 사용) || 퀵 정렬 | O(n log n) | 가장 빠름 (평균적) |
